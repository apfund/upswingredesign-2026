Editor Checklist: How to Reuse These Blocks

1. Copy the HTML snippet for the component you need into your page template or WordPress block.

2. Swap placeholder text (headlines, body copy, CTAs, prices) with your real content.

3. Update links and form actions - change href="#" to real URLs and connect forms to your email platform (GoHighLevel, etc.).

4. Replace placeholder images - logo wall items, instructor photos, and cover mocks should use real assets.

5. Pricing data attributes - for the pricing toggle, update data-monthly and data-annual on .c-plan-compare__price and .c-plan-compare__price-sub elements.

6. FAQ content - add or remove .c-faq__item blocks. Keep the first item with class .is-open for a good default state. Update chip labels and group IDs to match.

7. Quiz questions - update the quizData array in the script to change questions, options, and result segments.

8. Countdown timer - set your target date in the cdTarget variable in the script. For evergreen timers, use a fixed duration from page load instead.

9. Sticky CTA - change the scroll threshold (default 600px) and text/CTA in the HTML.

10. No em dashes - use hyphens (-) or "and" in all visible copy. This is a brand consistency rule across the design system.
